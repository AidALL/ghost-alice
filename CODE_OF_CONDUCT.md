# Code Of Conduct

Ghost-ALICE OS is built around careful reasoning, evidence, and recoverable execution. The same standard applies to project collaboration.

## Scope

This Code of Conduct applies within all project spaces, including the repository, issues, pull requests, discussions, code review, commit messages, and any other project-managed channel. It also applies when an individual is officially representing the project in public spaces.

## Expected Behavior

- Be respectful and direct.
- Critique ideas, patches, and evidence rather than people.
- Explain risks with enough context for others to verify them.
- Assume good intent, while still checking claims.
- Keep discussions focused on the project and its users.

## Unacceptable Behavior

- Harassment, threats, or personal attacks
- Discriminatory language or conduct
- Publishing private information without permission
- Sharing secrets, credentials, or exploit details in public channels
- Repeatedly derailing technical discussion after maintainers ask to stop

## Enforcement

Maintainers may moderate issues, pull requests, discussions, and other project spaces to keep collaboration productive and safe. Depending on severity, maintainers may warn, edit, hide, lock, or remove content, or block participation.

Security-sensitive behavior should follow [SECURITY.md](./SECURITY.md), not public discussion.

## Reporting Conduct Issues

Report conduct issues privately to the maintainers at `ceo@aidall.tech` with the subject prefix `[ghost-alice conduct]`. If the concern involves that contact, ask for a private alternate channel through the owner organization before sending sensitive details.

Maintainers will review reports as promptly and fairly as practical. Maintainers are obligated to respect the privacy and security of the reporter of any incident. Security-sensitive behavior should follow [SECURITY.md](./SECURITY.md), not this channel.
